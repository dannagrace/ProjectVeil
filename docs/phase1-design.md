# Project Veil Phase 1 设计稿

## 1. 产品定位

- 目标平台：手机网页优先，兼容微信小游戏/小程序生态容器部署。
- 核心玩法：大地图探索、资源拾取、点触交互、遭遇明雷进入回合制战斗。
- 竞品锚点：玩法结构高度参考《英雄无敌3》，但首期只保留最能验证留存的主循环。

## 2. 核心循环

1. 玩家进入区域房间，获得一个英雄和初始部队。
2. 英雄在迷雾地图上移动，消耗移动力并揭开视野。
3. 玩家拾取资源、访问建筑、触碰明雷怪物。
4. 若触碰怪物，服务端拉起战斗实例并按纯函数结算。
5. 战斗胜利后返回世界地图，发放经验/资源/地图控制权。
6. 日切时刷新移动力和部分地图物件。

## 3. Phase 1 功能拆分

### 必做

- 世界地图
  - 8x8 到 32x32 网格地图。
  - A* 寻路和点击移动。
  - 低成本遮罩式迷雾。
- 地图交互
  - 静态资源点。
  - 1 类周常建筑和 1 类临时增益建筑。
  - 明雷中立怪。
- 战斗系统
  - 双方兵堆。
  - 先攻排序。
  - 攻击、等待、防御三类指令里先做攻击与等待。
- 联机底座
  - 服务端权威世界状态。
  - Action -> Validate -> Reduce -> Broadcast 状态流。

### 暂缓

- 城镇经营。
- 多英雄编队。
- 深度技能树。
- 实时 PVP。
- 商城、联盟、公会。

## 4. 技术方案

### 客户端

- 建议引擎：Cocos Creator。
  - 原因：更适合 2D 地图、序列帧、微信生态发布。
  - 客户端职责仅限输入采集、本地插值、状态渲染。
- UI 框架：引擎原生 UI。
- 网络层：WebSocket，按房间订阅状态。

### 服务端

- 运行时：Node.js。
- 房间层：Colyseus。
- 核心原则：房间状态不可直接由表现层修改，所有改动都走共享 reducer。
- 安全原则：服务端只向玩家下发按视野裁剪后的 snapshot，不向客户端暴露全图状态。

### 共享模块

- `packages/shared`
  - `models.ts`：世界状态、战斗状态、Action 定义。
  - `map.ts`：地图生成、玩家私有迷雾、移动校验、资源拾取。
  - `battle.ts`：战斗回合排序、反击、确定性伤害结算。

## 4.1 本轮已落地的优化

- 迷雾从通用 Tile 状态中拆出，改为 `visibilityByPlayer` 私有状态。
- 服务端 snapshot 改为按玩家裁剪，隐藏区域不再下发资源和占位信息。
- 增加 `validateWorldAction` / `validateBattleAction`，服务端先校验再 reduce。
- 战斗增加每回合一次反击与确定性 RNG 游标，便于回放和前后端复现。
- 世界移动校验升级为 A* 路径搜索，不再使用简单曼哈顿距离近似。
- 玩家视图区分 `ownHeroes` 与 `visibleHeroes`，为后续遭遇战和侦察玩法留接口。
- 世界 Action 结果增加 `movementPlan` 和 `events`，客户端可直接按路径点序列播放移动。
- 接触中立怪时，世界房间会保留英雄停点，并同步返回 `battle.started` 事件和战斗快照。
- 战斗层增加胜负判定接口，服务端可在战斗结束后把结果回写到世界状态。
- 当前已实现攻击方胜利后移除中立怪并发放奖励，失败时保留世界原状并结束遭遇。
- 战斗指令已扩展为 `attack / wait / defend`，其中 `defend` 会在本轮内提高受击防御。
- 防守方胜利时，当前规则为英雄保留在战前位置、生命值衰减至 50%、本日移动力清零。

## 5. 小程序生态部署策略

- H5 首发
  - 用同一套共享逻辑跑网页版本，便于快速迭代与灰度。
- 微信小游戏适配
  - 客户端外壳迁移到 Cocos Creator 导出的小游戏工程。
  - 网络层保持 WebSocket 协议不变。
- 小程序导流
  - 可用小程序页承载活动页、登录页和社区页。
  - 核心对战建议跑在小游戏容器，不建议普通 WebView 承载高频战斗帧刷新。

## 6. 数值与内容生产

- 地图物件、兵种、建筑全部 JSON 化。
- 未来补一套 Web 配置中心，导出强类型 JSON 到共享模块。
- 配置变更流程：策划编辑 -> JSON 校验 -> 版本号提升 -> 热更新投放。
- 当前仓库已落地 `configs/phase1-world.json`，用于驱动地图尺寸、初始英雄、中立怪与资源生成概率。

## 7. 迭代顺序

1. 第 1 周：共享状态模型、世界地图移动、迷雾。
2. 第 2 周：资源拾取、中立怪触发、战斗纯函数。
3. 第 3 周：房间同步、断线重连、回放日志。
4. 第 4 周：客户端表现、首轮数值配置、首包上线验证。
