# Project Veil 开发路线图 (Roadmap)

本项目旨在以低成本验证《英雄无敌》Like 玩法在移动端的可行性。当前已完成基础原型，以下为后续开发计划。

---

## 🟦 Phase 1.5: 工程化补全与体验升级 (进行中)
**目标：** 提升原型质感，优化联机稳定性。

- [ ] **客户端引擎迁移**
  - [ ] 将 `apps/client` 逻辑层抽离，接入 Cocos Creator 或 Phaser 引擎。
  - [x] 实现高性能 Tilemap 瓦片地图渲染。
  - [x] 接入 Spine 或序列帧动画系统 (Idle/Walk/Attack/Hit)。
  - [x] 将 HUD / 地图 / 战斗面板 / 时间线拆成独立 Cocos 组件。
  - [x] 将战斗面板推进到可操作状态，支持目标选择与 `attack / wait / defend`。
- [ ] **网络与同步优化**
  - [x] 完善 `reconnection-storage`，支持秒级断线重连与状态回放。
  - [x] 实现客户端表现层预演 (Client-side Prediction)，消除点击移动的等待感。
- [ ] **视觉资产升级**
  - [x] 补齐 Cocos 侧迷雾边缘过渡层与地图交互反馈骨架。
  - [x] 将地图资源点/守军/敌方英雄的 Cocos 表现接到配置驱动标签层。
  - [x] 补齐低成本动态迷雾脉冲版本，作为后续 shader / Alpha 混合前的过渡实现。
  - [x] 为非 Tilemap 回退模式补齐独立迷雾覆盖层，先建立字符遮罩 + 透明度脉冲骨架。
  - [ ] 替换当前简易 SVG，采用“2.5D 高质感像素”风格。
  - [ ] 实现动态迷雾效果 (边缘羽化与 Alpha 混合)。
  - [x] 为地图物件增加交互反馈 (如拾取时的弹跳动画)。

---

## 🟩 Phase 2.0: 玩法深度与核心系统 (核心循环)
**目标：** 丰富战斗策略与地图交互，构建完整的 PVE/PVP 体验。

- [ ] **战斗系统扩展**
  - [ ] **技能系统**：支持主动技能（射击、施法）与被动 Buff（中毒、防御加成）。
  - [x] **待动序列可视化**：UI 展示回合行动顺序，增强策略预判。
  - [ ] **战场环境**：障碍物（阻挡格）、战场陷阱。
- [ ] **地图建筑功能化**
  - [ ] **招募所**：消耗资源招募部队。
  - [ ] **属性建筑**：访问后永久提升英雄攻击/防御/法力。
  - [ ] **资源产出点**：占领矿场，每日产生金币/木材。
- [ ] **AI 行为进化**
  - [ ] 实现中立怪巡逻、守卫、主动追击逻辑。

---

## 🟨 Phase 3.0: 持久化与成长体系 (长线运营)
**目标：** 实现跨会话数据存储，构建玩家成长动力。

- [ ] **MySQL 持久化落地**
  - [ ] 实现玩家账号系统与全局资源仓库。
  - [ ] 英雄存档：保存等级、技能、装备与当前带兵。
- [ ] **英雄成长系统**
  - [ ] 战斗经验值结算与自动升级逻辑。
  - [ ] 技能树系统：升级后可选择学习/强化特定技能。
- [ ] **世界/房间流转**
  - [ ] 实现“中心化大厅 (Lobby)”与“独立房间 (Instance)”的切换逻辑。

---

## 🛠️ 内容生产与自动化 (工具链)
**目标：** 降低配置成本，提升迭代效率。

- [ ] **配置中心增强 (Config-Center)**
  - [ ] **技能编辑器**：可视化配置 JSON 伤害公式与效果描述。
  - [ ] **地图生成器预览**：在保存 `phase1-world.json` 前可即时预览随机分布结果。
- [ ] **自动化测试**
  - [ ] 为 `packages/shared` 的战斗结算逻辑补齐 100% 覆盖率的单元测试。
  - [ ] 压力测试：模拟 100+ 并发房间的 CPU 占用监控。

---

## 🎨 视觉规范 (Technical Art)
- **风格**：2.5D 像素奇幻 / 扁平插画风。
- **色彩规范**：交互物高饱和，背景地形中低饱和。
- **动画帧率**：核心动画 12-15fps 像素风节奏。
