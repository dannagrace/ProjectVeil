# Project Veil 开发迭代计划

基于 Phase 1.5 现状、TODO.md 路线图与 MOE 专家建议，后续开发分 3 个迭代推进至 Phase 2.0。

---

## 迭代 A：Phase 1.5 收尾 + 性能基建

**目标：** 关闭 Phase 1.5，为 Phase 2.0 扩展扫清工程障碍。

### A-1 地图扩容支持
- [ ] `map.ts` 的 Tile 数据改用 TypedArray（Uint8Array）存储地形/迷雾，压缩 80%+ 序列化体积
- [ ] 实现九宫格区块化同步（Chunking），支持 32x32 及以上地图
- [ ] 将 `phase1-world.json` 的 mapSize 提升到 16x16 或 32x32 并验证性能

### A-2 迷雾视觉升级
- [ ] Cocos 侧实现 shader-based 迷雾（边缘羽化 + Alpha 混合），替换当前脉冲近似方案

### A-3 测试加固
- [ ] 为 `packages/shared` 战斗结算路径补齐单元测试至 90%+ 覆盖率
- [ ] 增加 32x32 地图下的寻路性能基准测试

---

## 迭代 B：Phase 2.0 核心 — 技能系统 + 建筑功能

**目标：** 大幅丰富战斗策略深度和地图交互内容。

### B-1 技能系统
- [ ] 在 `models.ts` 中扩展 `Skill` 数据模型（主动技能 / 被动 Buff）
- [ ] `battle.ts` 增加 `battle.cast` 动作和效果结算管线（伤害/治疗/状态叠加）
- [ ] 新增 `configs/skills.json` 技能配置表 + Config Center 技能编辑器
- [ ] Cocos 战斗面板增加技能选择 UI

### B-2 地图建筑功能化
- [ ] **招募所**：消耗资源招募部队（扩展 `hero.recruit` Action）
- [ ] **属性建筑**：访问后永久提升英雄攻击/防御/法力
- [ ] **资源产出点**：占领矿场，每日结算时产出金币/木材
- [ ] 在 `models.ts` 中扩展 `Building` 模型，`map.ts` 增加建筑交互逻辑

### B-3 战场环境
- [ ] 战斗地图增加障碍格（阻挡寻路）和陷阱格（踩踏触发效果）
- [ ] 扩展 `BattleState` 支持地形属性

---

## 迭代 C：Phase 2.0 完善 — AI + 工具链 + 压测

**目标：** PVE 体验闭环，工具链成熟。

### C-1 AI 行为系统
- [ ] 中立怪巡逻路径生成（基于 A* 的预定义路线）
- [ ] 守卫模式（玩家进入警戒范围触发拦截）
- [ ] 追击逻辑（视野内主动接近玩家英雄）

### C-2 内容生产工具
- [ ] Config Center 增加地图生成器预览（保存前即时可视化随机分布）
- [ ] 技能编辑器：可视化配置伤害公式和效果描述

### C-3 压力测试
- [ ] 模拟 100+ 并发房间，监控 CPU/内存
- [ ] 32x32 地图 8 人房间的网络带宽和帧率压测

---

## 优先级建议

如果资源有限，最推荐的起步顺序：

1. **A-1 地图扩容** — 解除 8x8 地图对后续所有玩法的限制，是 Phase 2 的前置依赖
2. **B-1 技能系统** — 策略深度的核心提升，直接影响留存验证
3. **B-2 建筑功能化** — 让地图探索有实质性回报，与技能系统形成正循环
